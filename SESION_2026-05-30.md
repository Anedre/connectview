## 1. WhatsApp status events (#14) — CERRADO como "no-relay"
- Cliente usa **AWS End User Messaging Social** (WABA "Novasys-Amazon",
  wabaId 2370402863397858, número +1 555-784-9134, qualityRating GREEN).
- Una WABA tiene **UN solo event destination**; el de ellos es la **instancia
  de Connect** (así los WhatsApp entrantes se vuelven chats de agente).
- Para capturar delivered/read/failed habría que repuntear ese destination a un
  SNS propio y **re-inyectar** el inbound con `SendChatIntegrationEvent`.
- **PROBADO:** esa API está **bloqueada al service principal** de End User
  Messaging — devuelve `AccessDenied` incluso con `AdministratorAccess`. Por
  lo tanto el relay rompería el inbound de los agentes. NO se hace.
- **Decisión del usuario:** usar **Meta Business Manager** para ver
  entregado/leído. El reporte HSM de Vox queda en "enviado + volumen".
- Se ajustó `HsmOutboundReport.tsx` para ser honesto (sin columnas
  delivered/read en 0; nota que apunta a Meta BM).
- Detalle completo en memoria: `project_connect_streams_gotchas.md` #3.

## 2. Fix build-breakers "WHISPER"

## 3. #16 BOT BUILDER / SALESBOT (lo más grande) — COMPLETO (builder+runtime)
El equivalente al Salesbot de Kommo. Constructor visual de flujos.

### Modelo de datos
- `src/lib/botFlow.ts` — tipos `Bot/BotNode/BotEdge`, catálogo `NODE_KINDS`
  (fuente única que alimenta paleta + nodo + inspector). **13 tipos de paso:**
  start, message (con botones tipo reply/url/phone), **list** (WhatsApp, hasta
  10 filas), question, condition, template (HSM con editor de variables), delay,
  set_field, handoff (cola+prioridad+asignar+nota), internal_note, webhook,
  jump, stop, **ai_agent**. Helpers: `makeNode`, `validateBot`, `defaultBot()`.
- `BOT_TEMPLATES` (6 flujos predefinidos): En blanco, Bienvenida+menú,
  Calificación de lead, Preguntas frecuentes (lista), Agente IA, Reactivación.

### UI del builder (dep nueva: `@xyflow/react` 12.10.2 = react-flow)
- `src/components/bots/StepNode.tsx` — nodo custom: banda de encabezado con
  degradado, chip de icono, badge numerado (estilo Kommo), pills inline
  "+ Botón de acción / + Enlace web", ramas como filas con conector a la derecha.
- `src/components/bots/FlowBuilder.tsx` — canvas (paleta izq + react-flow centro
  con minimapa/controles + inspector der generado del catálogo + toolbar
  nombre/estado/validación/Guardar/**Probar**). Editores: ButtonsEditor (con
  tipo+valor), VarListEditor, ListRowsEditor.
- `src/components/bots/builderCtx.tsx` — contexto para que el nodo se edite a
  sí mismo (numberOf/updateNodeData/selectNode).
- **GOTCHA react-flow:** doble React por instalar con Vite corriendo → agregar
  `resolve.dedupe:["react","react-dom"]` + `optimizeDeps.include:["@xyflow/react"]`
  en vite.config + reiniciar. Edges no dibujan / screenshot cuelga en el preview
  HEADLESS (rAF pausado) — verificar en Chrome real.
